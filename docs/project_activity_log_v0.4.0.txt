4.2  ARCHITECTURE DECISIONS
--------------------------------------------------------------------------------

DECISION: streaming.py (direct loop) vs base_agent.py (LangGraph)
  OPTIONS PRESENTED:
    A) LangGraph StateGraph — full agent with typed state, complex branching
    B) streaming.py agentic loop — simpler, direct tool-use loop, up to 5 iter
  CHOSEN: Option B (streaming.py) — simpler, no state machine overhead,
          sufficient for linear tool-use chat pattern

DECISION: Tool injection pattern
  PROBLEM: Tools need AsyncSession (DB) but registry must dispatch by name
  SOLUTION: Create tool instances per-request with session injected in __init__
            Build a new ToolRegistry per request via build_tool_registry(db)

DECISION: Chat persistence strategy
  PROBLEM: Must save ChatMessage + AIAuditLog after SSE stream ends
  SOLUTION: Wrap stream generator in try/finally — persist in finally block
            (runs even if client disconnects mid-stream)
