7.2  RECOMMENDATIONS & DECISIONS
--------------------------------------------------------------------------------

RECOMMENDATION: C++ Build Tools installation
  OPTIONS:
    A) Install Microsoft C++ Build Tools (~4-6 GB) — required for chromadb
    B) Use pre-built chromadb wheel — not readily available
  CHOSEN: Option A — guided user to download from visualstudio.microsoft.com
  USER QUESTION: "Which option — AMD64 or ARM64?"
  ANSWER: Checked via PROCESSOR_ARCHITECTURE env var; AMD64 confirmed

RECOMMENDATION: cl.exe not in PATH
  ISSUE: After VS Build Tools install, 'cl' not recognised in regular terminal
  RESOLUTION: Informed user that pip finds compiler via Windows Registry
              automatically; direct pip install works without adding cl to PATH

RECOMMENDATION: ChromaDB version mismatch
  ISSUE: Python client 0.5.23 calls /api/v2 endpoints; Docker image 0.5.0
         only has /api/v1 — results in 404 Not Found error
  OPTIONS:
    A) Downgrade Python client to match Docker server (0.5.0)
    B) Upgrade Docker image to match Python client (0.5.23)
  CHOSEN: Option B — updated docker-compose.yml; cleaner, keeps server/client in sync

DECISION: Claude hallucinating placeholder UUID
  ISSUE: Claude generated "priya-sharma-uuid" as tool input instead of real UUID
  ROOT CAUSE: System prompt didn't tell Claude the actual client UUID;
              `streaming.py` only injected client_id if NOT already in tool_input
              (so when Claude supplied a wrong value, it wasn't overridden)
  FIX APPLIED (two-pronged):
    1. streaming.py: Always force-override client_id in tool_input (remove `if` guard)
    2. streaming.py + chat.py: Embed actual UUID in system prompt with explicit
       instruction "use the client_id provided above — never guess"

DECISION: RAG graceful degradation
  DESIGN CHOICE: If ChromaDB is unreachable, chat continues without RAG context
                 (rather than failing the entire request)
  IMPLEMENTATION: _is_rag_available() cached singleton check; try/except around
                  RAG injection; falls back to base system prompt only

RAG ENGINE BUILT (backend/app/rag/):

  embedder.py:
    _get_chroma_client() — HttpClient to Docker ChromaDB
    _get_collection() — gets/creates collection with SentenceTransformerEmbeddingFunction
    _detect_section() — identifies section headers in knowledge docs
    _chunk_text() — splits at paragraph/sentence boundaries (500 chars, 100 overlap)
    _load_doc() — yields (chunk_text, section, chunk_index) per document
    index_knowledge_docs() — clears + re-indexes all .txt files; batch upsert 500/batch

  retriever.py:
    _get_collection() — lru_cache singleton ChromaDB collection
    QUERY_TYPE_TO_SOURCES mapping — focuses retrieval on relevant docs per query type:
      tax → [indian_tax_guide, sebi_ia_regulations]
      retirement → [retirement_planning_guide, indian_tax_guide]
      portfolio → [mutual_funds_guide, goal_based_investing]
      investment_advice → [mutual_funds_guide, goal_based_investing, sebi_ia_regulations]
      market → [mutual_funds_guide]
      behavioral → [goal_based_investing, retirement_planning_guide]
      general → [] (search all)
    retrieve() — semantic search with optional source filter; returns score + metadata
    format_context_for_prompt() — filters by min_score (0.3), truncates at 3000 chars
    get_rag_context() — async convenience wrapper (retrieve + format in one call)

  index.py — CLI entry point for indexing

  __init__.py — public API exports
