8.2  Recommendations & Decisions
--------------------------------------------------------------------------------
Decision: Phase 10 demo prep ran before bugs were discovered.
  → Health check, seed verify, RAG index, login tests all passed first.
  → Bugs found during live API testing.

Decision: Chat bug diagnosis required 3 iterations.
  1. First suspected wrong field name (message vs query) — wrong, validation error
     was from test payload, not root cause.
  2. Then found correct payload still errored — needed actual exception text.
  3. Added debug logging to streaming.py → found tool_execution_failed with
     UnicodeEncodeError on ₹ symbol.
  4. Separately found ANTHROPIC_API_KEY empty in backend config — pydantic-settings
     env var priority bug.

Decision: env_ignore_empty=True chosen over other approaches.
  Alternatives considered:
  - Set ANTHROPIC_API_KEY as system env var permanently → fragile, machine-specific
  - Pass key explicitly to uvicorn subprocess → works but manual each session
  - env_ignore_empty=True in config → proper fix, works always, no manual steps

Decision: ASCII encode tool summaries (not full result).
  Only `input_summary` and `output_summary` strings (for logging only) get encoded.
  The actual tool result dict passed to Claude is unchanged — ₹ symbol preserved
  in JSON response to Claude (JSON handles Unicode natively).

Bug Fix 1 — GoalsView (frontend/src/pages/investor/GoalsView.jsx):
  Problem: data?.data || [] gets object {goals:[], retirement_readiness:{}}
           → goals.map() fails silently, nothing renders
  Problem: goal.progress_pct undefined → progress bar = 0
  Problem: goal.target_year undefined → "Target: undefined"

  Fix: goal.feasibility_score for progressPct
  Fix: new Date().getFullYear() + goal.years_remaining for targetYear

Bug Fix 2a — config.py (backend/app/core/config.py):
  Problem: Claude Code sets ANTHROPIC_API_KEY='' in system environment.
           pydantic-settings v2 reads env vars before .env file.
           Empty string IS a value → overrides .env → API key blank in backend.
  Fix: env_ignore_empty=True in SettingsConfigDict → empty env vars ignored,
       falls through to .env value.

Bug Fix 2b — base_tool.py (backend/app/ai/base_tool.py):
  Problem: log_ai_call() receives str(result) containing ₹ (U+20B9).
           structlog writes to Windows console (cp1252 encoding).
           cp1252 cannot encode ₹ → UnicodeEncodeError.
           BaseTool.__call__ catches Exception → raises AIToolExecutionError.
           streaming.py catches AIToolExecutionError → yields SSE error event.
  Fix: .encode('ascii','replace').decode('ascii') on input_summary + output_summary.
       Replaces ₹ with '?' in log strings only. Tool result to Claude unchanged.
