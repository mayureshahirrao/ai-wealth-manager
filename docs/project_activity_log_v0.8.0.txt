9.2  RECOMMENDATIONS & DECISIONS
──────────────────────────────────
  a) Docker disk space (~14 GB growth)
     Cause:  WSL2 ext4.vhdx virtual disk grows on write but never auto-shrinks.
             Docker image layer cache compounds this.
     Fix:    Prune unused images (docker image prune -a). Compact vhdx with
             diskpart (select vdisk, compact vdisk).
     Decision: Document only; user to run when needed.

  b) ChromaDB v0.5 API change
     Cause:  ChromaDB ≥0.5 requires UUID for direct collection endpoint.
             Name-based GET /collections/{name} returns InvalidCollection.
     Fix:    Use GET /collections to list and retrieve UUID, then use
             GET /collections/{uuid}/count.
     Decision: Updated README verify commands to use UUID pattern.

  c) Axios timeout for AI endpoints
     Problem: Default 30s too short for Claude + large max_tokens.
     Options: (1) Increase max_tokens (2) Increase timeout (3) Both
     Decision: Both — reduced max_tokens 1500→800 AND increased timeout 30s→120s.
     Rationale: Defense in depth; faster response + tolerant frontend.

  d) Compliance doc type mismatch
     Problem: Frontend sent 'DISCLOSURE_DOC', backend enum is 'sebi_disclosure'.
     Decision: Fix frontend — backend enum is the source of truth.

  e) Developer guide format
     Problem: docx skill wrote to sandboxed temp dir, not project folder.
     Fix:    Write JS script directly to project, run with explicit NODE_PATH
             pointing to npm global modules dir on Windows.
     NODE_PATH: C:/Users/Mayuresh/AppData/Roaming/npm/node_modules
